# HunterDog - ระบบจัดการร้านหมาล่า

ระบบหลังบ้านและหน้าสั่งอาหารสำหรับร้านหมาล่า สร้างด้วย **Next.js** + **PostgreSQL** + **Prisma**

## บทบาทผู้ใช้

| บทบาท | เข้าผ่าน | หน้าที่ |
|--------|----------|---------|
| **Admin** | `/admin/login` | จัดการสาขา, เมนู, พื้นที่ส่ง, เบอร์พนักงาน, ค้นหาลูกค้า |
| **Staff** | `/staff/login` (เบอร์โทร) | ดูออเดอร์หน้าเดียว, อัปเดตสถานะตาม role |
| **ลูกค้า** | `/order` | เลือกสาขา, ชมเมนูได้โดยไม่ต้อง login, สั่งอาหาร (login ด้วยเบอร์โทร + ชื่อ ตอนเลือกสินค้า) |

## เส้นทางหลักฝั่งลูกค้า

| Route | หน้าที่ |
|-------|---------|
| `/order` | เลือกสาขา (ค้นหาได้, เห็นสถานะเปิด/ปิด) |
| `/{brandCode}/{branchCode}` เช่น `/malawaiwai/klong6` | ลิงก์ตรงเข้าสาขา |
| `/order/store/{branchId}` | เมนูสาขา + รายละเอียดสินค้า (ความเผ็ด/ซอส/น้ำจิ้ม/จำนวน/หมายเหตุ) |
| `/order/checkout` | สรุปคำสั่งซื้อ: รับที่ร้าน/จัดส่ง, เวลารับ, หมายเหตุ, วิธีชำระเงิน |
| `/order/confirmation/{orderId}` | สั่งซื้อสำเร็จ + เลขออเดอร์ + timeline สถานะ |
| `/order/history` | ประวัติ + filter ตามสถานะ |
| `/order/history/{orderId}` | รายละเอียด + ยกเลิก (เฉพาะรอร้านรับ) + สั่งซื้ออีกครั้ง |

## สถานะออเดอร์

- 🟡 รอร้านรับออเดอร์ (`WAITING_FOR_STORE_ACCEPTANCE`) — ลูกค้ายกเลิกได้เฉพาะสถานะนี้
- 🟠 กำลังเตรียม (`PREPARING`)
- 🩵 พร้อมรับสินค้า (`READY_FOR_PICKUP`) — เฉพาะออเดอร์รับที่ร้าน
- 🔵 รอการจัดส่ง (`READY_FOR_DELIVERY`) — เฉพาะออเดอร์จัดส่ง
- 🟣 กำลังจัดส่ง (`DELIVERING`)
- 🟢 เสร็จสิ้น (`COMPLETED`)
- ⚪ ยกเลิก (`CANCELLED`)

**คนขาย**: รับออเดอร์ → เตรียม → พร้อมรับ/รอส่ง (+ ยกเลิกออเดอร์ที่ยังไม่รับ)  
**คนส่ง**: รอการจัดส่ง → กำลังจัดส่ง → เสร็จสิ้น (เฉพาะออเดอร์จัดส่ง)

## เริ่มใช้งาน

### 1. ติดตั้ง dependencies

```bash
npm install
```

### 2. ตั้งค่า PostgreSQL

แก้ไข `.env`:

```env
DATABASE_URL="postgresql://postgres:postgres@localhost:5432/hunterdog?schema=public"
JWT_SECRET="change-me-to-a-long-random-string"
```

### 3. สร้างฐานข้อมูล

```bash
npm run db:push
npm run db:seed
```

### 4. รัน dev server

```bash
npm run dev
```

เปิด [http://localhost:3000](http://localhost:3000)

## โครงสร้างหลัก

```
src/app/
  admin/          # หลังบ้าน admin
  staff/          # หน้าพนักงาน (ออเดอร์บอร์ด)
  order/          # หน้าลูกค้าสั่งอาหาร
  api/            # REST API
prisma/
  schema.prisma   # โมเดลฐานข้อมูล
```

## สมมติฐานที่ใช้ในเวอร์ชันนี้

1. **ลูกค้าและพนักงาน** login ด้วยเบอร์โทร (ไม่มี OTP) — ลูกค้าต้องกรอกชื่อด้วย
2. **Admin** login ด้วย username/password
3. **เมนูเป็นของแต่ละสาขา** พร้อมหมวดหมู่และตัวเลือก (ความเผ็ด/ซอส/น้ำจิ้ม)
4. **Staff board / หน้า tracking** refresh อัตโนมัติ (ยังไม่มี WebSocket)
5. **วิธีชำระเงินเป็นแบบ manual** (เงินสด/โอน/บัตร) — ยังไม่เชื่อม payment gateway
6. **รูปภาพร้าน/สินค้าใน seed เป็น placeholder** จาก picsum.photos
